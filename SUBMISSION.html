Autonomous Compliance Agent powered by Elasticsearch & LangChain

1. Project Description

The Problem

Global financial institutions face billions of dollars in fines annually due to non-compliance with Anti-Money Laundering (AML) and Know Your Customer (KYC) regulations. These regulations are vast, complex, and constantly changing. Compliance officers spend countless hours manually searching through thousands of PDF pages, often missing critical nuances. Standard chatbots fail here because they hallucinate laws or cannot access private, up-to-the-measure regulatory documents.

The Solution

JurisLens is an autonomous agent designed to bridge the gap between legal rigor and AI speed. Unlike a passive chatbot, it is equipped with specialized tools:

  • Search & Retrieve: Actively queries a secure Knowledge Base using Semantic Search to find exact legal clauses.
  • Calculate Risk: Switches modes to perform deterministic financial risk assessments.
  • Cite Sources: Provides answers grounded in actual text, reducing liability.
Architecture & Tech Stack:
  • Elasticsearch Vector Store: The semantic "brain" storing thousands of regulatory embeddings.
  • LangChain Agents: Uses OpenAI Functions Agent for intelligent tool routing.
  • RAG Pipeline: Automated ingestion (Chunking -> Embedding -> Indexing).

3 Features We Liked & Challenges

  1. Elasticsearch Speed: The retrieval speed for vector search was phenomenal. Even with dense legal texts, Elastic returned relevant chunks in milliseconds.
  2. Hybrid Tool Use: We loved how the Agent seamlessly combined PDF search data with a Python-based risk calculator in a single conversation turn.
  3. Challenge (Solved): Prompt Engineering for Tools. Getting the agent to consistently call the RiskCalculator with the correct JSON schema was tricky, but moving to Pydantic-based structured tools solved it.
